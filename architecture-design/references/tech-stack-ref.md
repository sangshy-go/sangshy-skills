# 技术选型速查表

> 仅供参考，具体选型应根据项目需求判断，每个选型必须说明理由。

## 项目类型 → 选型关注点

| 项目类型 | 典型场景 | 选型关注点 |
|---------|---------|-----------|
| **Web 全栈** | SaaS 平台、管理系统、电商 | 前后端框架、数据库、缓存、CDN |
| **纯后端服务** | API 网关、微服务、数据处理 | 语言/框架、消息队列、数据库、缓存 |
| **CLI 工具** | 命令行工具、开发脚手架 | 语言选择、打包分发、交互体验 |
| **桌面应用** | Tauri/Electron 应用 | 跨端框架、系统 API 访问、打包 |
| **移动应用** | Android/iOS 原生或跨端 | 框架选择、推送、离线存储 |
| **数据处理** | ETL、爬虫、数据分析管道 | 数据处理框架、调度、存储 |

## 推荐方案

| 场景 | 推荐方案 | 说明 |
|------|---------|------|
| Web 全栈（中小型） | React + Node/Express + PostgreSQL | 成熟稳定，生态丰富 |
| Web 全栈（快速原型） | Next.js + SQLite/Turso | 全栈框架，部署简单 |
| Web 全栈（大型） | React + Java/Spring Boot + PostgreSQL | 企业级，高并发 |
| 高并发 API | Go + Redis + PostgreSQL | 性能优异 |
| CLI 工具 | Node.js (commander) 或 Go | Go 编译为单二进制更方便分发 |
| 桌面应用 | Tauri (Rust+Web) | 体积小，性能好 |
| 数据处理管道 | Python + pandas/Airflow | 数据生态最完善 |
| 移动端跨平台 | React Native 或 Flutter | 按团队技术栈选择 |

## 选型输出格式

根据项目类型选择适用的部分：
- 语言/运行时 + 理由
- 核心框架 + 理由
- 数据存储 + 理由
- 认证/安全 + 理由
- 测试框架 + 理由
- 部署方案 + 理由

**Web 全栈额外**：前端框架 + 理由、UI 库 + 理由、状态管理 + 理由、后端框架 + 理由、ORM + 理由
**CLI/桌面/移动额外**：打包/分发方案 + 理由、系统级能力（文件、网络、通知等）+ 理由
