# 架构设计 Skill

## 工作流程

### Step 1：读取并分析 PRD
读取 `docs/prd.md`。如 PRD 不存在或内容过少（< 5 行），先与用户确认需求要点或直接调用 requirement-analysis 补充。
提取：
- 核心功能模块
- 数据规模预估（用户量、数据量、并发量）
- 性能/安全/可用性要求
- 特殊技术约束

### Step 2：判断项目类型
根据 PRD 判断项目类型，读取 `references/tech-stack-ref.md` 进行技术选型。

### Step 3：技术选型
基于项目类型和 PRD 要求，给出**带理由**的技术选型（每个选型必须说明"为什么"）。

### Step 4：系统架构设计
输出系统整体架构，使用 Mermaid 绘制架构图、数据流图。
包含模块划分表：模块名称、职责、主要功能。

### Step 5：接口设计
- Web/API 项目：RESTful API 或 gRPC 接口清单表
- CLI/桌面项目：命令行参数规范或 IPC 通信接口
- 数据处理项目：数据输入输出规范、调度触发方式
包含：基础规范、接口清单表、关键接口详细设计（请求体/响应体）。

### Step 6：数据库设计
（如项目无持久化需求则跳过）
- 关系型数据库：ER 图（Mermaid）、表结构（字段名/类型/必填/默认值/描述）、索引设计
- NoSQL/缓存：集合/Key 设计、数据结构说明
- 文件存储：目录结构规范、文件命名规则

## 约束
- 产出保存到 `docs/architecture.md`
- 技术选型必须说明理由，禁止无理由堆砌
- 架构图必须用 Mermaid 绘制，不用 ASCII art
