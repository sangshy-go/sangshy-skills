# 需求分析 Skill

## 工作流程（必须按顺序执行）

### Step 1：需求澄清
在开始任何产出前，先向用户提问澄清：
- 核心目标：这个产品/功能要解决什么问题？
- 目标用户：谁会使用？有什么特征？
- 核心场景：用户在什么情况下使用？
- 功能边界：哪些在 MVP 内？哪些不在？
- 成功指标：如何衡量功能做成功了？

输出：「需求澄清报告」

### Step 2：竞品研究
使用 WebSearch 和 fetch 工具搜索 3～5 个竞品：
- 各竞品核心功能对比
- 差异化优势和劣势
- 用户反馈和痛点

输出：「竞品分析报告」

### Step 3：PRD 起草
读取 `references/prd-template.md`，严格按照模板生成 PRD 初稿。

### Step 4：内部评审挑错
用研发/测试/用户三个视角对 PRD v1.0 自我评审。
输出：「评审意见清单」（问题描述 + 影响程度 + 建议修改方向）

### Step 5：终稿收敛
根据评审意见修改 PRD，输出最终版本。

## 约束
- 产出保存到 `docs/prd.md`
- PRD 必须包含版本规划（v1.0 MVP 明确标注 P0/P1/P2）
- 用户交互设计必须根据项目类型选择对应描述方式（见 prd-template.md）
