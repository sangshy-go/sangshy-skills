# 桑夏自动写作流程

自动化写作助手：先搜索验证，再动笔写作。

## 核心流程

```
1. 接收命题 → 2. 搜索素材 → 3. 整理信息 → 4. 按风格写作
```

## 步骤详解

### Step 1: 接收命题

用户给出写作主题，例如：
- "什么是 Agent Harness？"
- "RAG 技术原理"
- "TypeScript 泛型详解"

**确认命题范围**：
- 主题是否清晰？
- 是否需要聚焦特定角度？
- 目标读者是谁？

### Step 2: 搜索素材

**搜索顺序**：

1. **官方文档** (优先级最高)
   - 搜索：`<主题> official documentation`
   - 搜索：`<主题> docs`

2. **热门文章** (3-5 篇)
   - 搜索：`<主题> tutorial guide 2025 2026`
   - 搜索：`<主题> explained`
   - 搜索：`<主题> best practices`

3. **深度文章** (1-2 篇)
   - 搜索：`<主题> deep dive`
   - 搜索：`<主题> architecture internals`

**搜索工具**：
```bash
# 使用 tavily-search
node {baseDir}/../tavily-search/scripts/search.mjs "<query>" -n 10 --deep
```

**提取内容**：
```bash
# 用 extract 获取全文
node {baseDir}/../tavily-search/scripts/extract.mjs "<url>"
```

### Step 3: 整理信息

**信息来源表**：

| 来源 | 类型 | 核心观点 | 引用位置 |
|------|------|----------|----------|
| URL 1 | 官方文档 | 定义、API | 基础概念部分 |
| URL 2 | 热门教程 | 实战示例 | 实战部分 |
| URL 3 | 科技博客 | 架构分析 | 原理部分 |

**去重和验证**：
- 交叉验证多个来源的说法
- 优先采用官方文档的定义
- 标注有争议或不确定的信息

### Step 4: 按风格写作

**默认风格：原理科普**

结构模板：
```
1. 问题驱动开场（引用块提出核心问题）
2. 官方定义（引用原始来源）
3. 架构图 + 文字解释
4. 核心组件/原理分点说明（最多 2 层序号）
5. 工作流程/示例
6. 总结 + 参考资料
```

**风格要求**：
- 无 emoji（除引用外）
- 序号清晰（1 → 1.1 → 1.1.1，最多 2 层）
- 表格对比信息
- ASCII 架构图配文字解释
- 字数统计和阅读时间

**其他可选风格**：
- 教程风格：Step-by-Step 实战
- 评测风格：对比分析 + 结论
- 新闻风格：事件 + 影响

## 输出格式

文章保存到：
```
articles/YYYY-MM-DD-<主题>.md
```

## 写作原则

> **先搜索，后写作。不知道的不编造。**

1. 所有核心概念必须有来源
2. 定义优先引用官方
3. 不确定内容明确标注
4. 参考资料列表完整

## 快速开始

```bash
# 命令行参数：主题 [风格]
node scripts/write.js "Agent Harness 原理" "原理科普"
```
