# 原理科普风格指南

> 适用于：技术原理、架构讲解、概念解析类文章

## 核心原则

**清爽、准确、有依据**

- 不编造：所有核心概念必须有来源
- 不啰嗦：能用图表说明的，不写长段解释
- 不花哨：无 emoji（除引用外），保持专业感

## 文章结构

### 1. 问题驱动开场

用引用块提出核心问题，引发读者思考：

```markdown
> 为什么同样的 AI，装了 Skills 后能力突飞猛进？
>
> 答案在于 Skills 的三层加载机制。
```

### 2. 官方定义

引用权威来源的定义：

```markdown
**OpenClaw 官方文档**：

> Skills 是模块化、自包含的功能包...
>
> 来源：[链接]
```

### 3. 架构图 + 解释

ASCII 图展示工作原理：

```markdown
┌─────────────────────────────────────┐
│  用户请求                           │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  L1: Metadata (始终在上下文)        │
└──────────────┬──────────────────────┘
               │ 匹配成功
               ▼
┌─────────────────────────────────────┐
│  L2: SKILL.md (触发后加载)          │
└─────────────────────────────────────┘
```

**必须配文字解释**：

```markdown
**架构图说明**：

| 层级 | 内容 | 加载时机 | 设计目的 |
|------|------|---------|---------|
| L1 | name + description | 始终加载 | 快速匹配 |
| L2 | SKILL.md 全文 | 触发后加载 | 提供完整指令 |
```

### 4. 核心原理分点说明

**序号规则**：
- 最多 2 层：`1 → 1.1` 或 `一 → 1`
- 不跳级：不要 `1 → 1.1.1.1`

```markdown
## 1. 核心组件

### 1.1 上下文管理

内容...

### 1.2 工具编排

内容...
```

### 5. 对比表格

用表格展示差异：

```markdown
| 方式 | 计算 | 结果 |
|------|------|------|
| 全文加载 | 100 skills × 5KB | 500KB |
| 三层加载 | 1 触发的 skill × 5KB | 5KB |
```

### 6. 总结 + 参考资料

```markdown
## 总结

核心要点：
1. 要点 1
2. 要点 2
3. 要点 3

## 参考资料

- [标题](URL)
- [标题](URL)
```

## 格式要求

| 项目 | 要求 |
|------|------|
| Emoji | 不用（除引用外） |
| 序号层级 | 最多 2 层 |
| 架构图 | 必须有 + 文字解释 |
| 表格 | 至少 1 个对比表 |
| 来源标注 | 核心概念必须有 |
| 字数统计 | 文章末尾标注 |

## 检查清单

写作完成后检查：

- [ ] 开场有引用块提出问题
- [ ] 官方定义有来源链接
- [ ] 架构图有文字解释
- [ ] 序号不超过 2 层
- [ ] 至少 1 个对比表格
- [ ] 所有核心概念有来源
- [ ] 无 emoji（除引用外）
- [ ] 末尾有字数统计
- [ ] 末尾有参考资料列表
